# Push

智能部署：自动检测代码变更，只部署修改的部分。

## 触发条件

当用户说以下内容时，自动执行此命令：
- "前端改好了，推送到服务器"
- "更新后端到服务器"
- "帮我部署代码"
- "推送到生产环境"
- "同步代码到服务器"
- "更新线上代码"
- "后端改完了"
- "前端修改完成了"
- "部署一下"

## 执行步骤

1. 检测 server/dist/ 和 client/dist/ 的修改时间
2. 对比上次部署记录，判断是否需要更新
3. 只部署有变更的部分

```bash
cd f:/uniapp-project/deploy && node cli.js push
```

## 智能检测逻辑

| 检测结果 | 操作 |
|----------|------|
| 只有后端变更 | 只部署后端 |
| 只有前端变更 | 只部署前端 |
| 都有变更 | 部署全部 |
| 无变更 | 提示无需部署 |

## 相关命令

- `/push-backend` - 强制推送后端
- `/push-frontend` - 强制推送前端
- `/deploy` - 完整部署前后端
- `/status` - 查看服务状态
